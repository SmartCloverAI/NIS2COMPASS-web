# Confidentiality Labels

Every artifact should have a confidentiality label.

| Label | Meaning | Public repo allowed |
| --- | --- | --- |
| Public-safe | Safe for public release. | Yes |
| Funder-facing | Intended for funder review, may contain non-public detail. | No, unless explicitly cleared. |
| Internal | Intended only for project team use. | No |
| Pilot-sensitive | Could expose a pilot organization, person, system, weakness, log, or business detail. | No |
| Synthetic | Fake data created for examples or demos. | Yes, if clearly marked. |
