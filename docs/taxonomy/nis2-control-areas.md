# NIS2 Control Areas

This public taxonomy groups NIS2 readiness work into practical control areas. It is a learning structure, not a legal interpretation.

| Control area | Plain meaning | Example evidence |
| --- | --- | --- |
| Governance | Who owns cybersecurity decisions and how they are reviewed. | Responsibility matrix, decision log, policy approval record. |
| Risk management | How risks are identified, scored, treated, and reviewed. | Risk register, treatment plan, review notes. |
| Incident handling | How incidents are detected, escalated, handled, and reported. | Incident playbook, exercise notes, reporting workflow. |
| Business continuity | How important services keep operating or recover after disruption. | Backup test, recovery plan, continuity exercise. |
| Supply-chain security | How suppliers and dependencies are assessed. | Supplier inventory, security questionnaire, dependency review. |
| Technical protection | How systems, networks, identities, and data are protected. | Configuration baseline, patch record, access review. |
| Monitoring | How events and alerts are collected and reviewed. | Monitoring source inventory, sanitized alert summary. |
| Vulnerability handling | How weaknesses are found, prioritized, fixed, and verified. | Vulnerability register, remediation evidence, retest record. |
| Awareness | How people are trained and behavior is improved. | Training material, attendance list, phishing metrics. |
| Evidence integrity | How evidence files are protected against silent change. | SHA-256 hash manifest, approval log. |
