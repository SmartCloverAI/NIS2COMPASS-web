# Glossary

| Term | Meaning |
| --- | --- |
| API | Application Programming Interface. A defined way for software systems to exchange data. |
| CYberSynchrony | The larger EU-funded cybersecurity project that funds and frames NIS2COMPASS. |
| CVSS | Common Vulnerability Scoring System. A standard method for scoring vulnerability severity. |
| Deliverable | A promised project output, such as a report, template, schema, dataset, public playbook, or website page. |
| Evidence | A file or record that proves a project claim. |
| Evidence Graph | A structured model that links controls, evidence, approvals, findings, remediation, and reports. |
| FSTP | Financial Support to Third Parties. EU project funding distributed by a larger project to selected external projects. |
| Indicator of Compromise | A clue that may indicate malicious activity, such as a suspicious IP address, file hash, domain, or URL. |
| JSON | JavaScript Object Notation. A structured machine-readable text format. |
| JSONL | JSON Lines. A file format where each line is one JSON record. |
| MISP | Malware Information Sharing Platform. An open-source platform for sharing threat intelligence. |
| MITRE ATT&CK | Public knowledge base of attacker tactics and techniques. |
| NIS2 | The EU Network and Information Security Directive 2, formally Directive (EU) 2022/2555. |
| OCMT | Open Call Management Team. The CYberSynchrony team managing open-call projects and checkpoints. |
| OWASP | Open Worldwide Application Security Project. A public community that publishes application security guidance. |
| Public-safe | Safe to publish because sensitive data, personal data, secrets, and exploitable details are removed. |
| Ratio1 RedMesh | Optional supporting tool for authorized repeatable or distributed validation, when approved and safely scoped. |
| Rules of Engagement | Written authorization and boundaries for security testing. |
| STIX | Structured Threat Information Expression. A standard format for cyber threat intelligence. |
| TAXII | Trusted Automated Exchange of Intelligence Information. A protocol for exchanging threat intelligence over HTTPS. |
| VA/PT | Vulnerability Assessment and Penetration Testing. Finding weaknesses and safely validating selected weaknesses under written authorization. |
| Wazuh | Open-source security platform used for monitoring, security event management, file integrity monitoring, vulnerability detection, and compliance support. |
| Suricata | Open-source network threat detection engine. |
